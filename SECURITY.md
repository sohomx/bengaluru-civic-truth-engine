# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for suspected secrets, private data exposure,
or a vulnerability that could put users at risk.

Send the report privately to the repository owner with:

- A short description of the issue.
- The affected file, route, command, or workflow.
- Steps to reproduce, if safe to share.
- Whether any private, account-linked, or non-public civic data may be involved.

## Data Safety Scope

This project is designed around public-safe civic records. It should not publish
private complaint tracking data, OTP/login material, account-linked identifiers,
API keys, credentials, or private citizen details.

The public demo is not an official government service and does not prove live
issue status, field condition, agency action, or resolution.
